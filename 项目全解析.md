# TOP投诉小区隐患整治系统 - 全项目解析

## 一、项目概述

**项目名称**：TOP投诉小区隐患整治系统
**技术栈**：React 18 + TypeScript + Vite + Tailwind CSS + Zustand
**项目用途**：面向电信运营商的投诉小区隐患工单管理平台，支持地市/区县两级管理员协同处理隐患工单。

## 四、核心模块详解

### 4.1 类型定义 (`src/types/index.ts`)

| 类型/枚举 | 说明 |
|-----------|------|
| `UserRole` | 用户角色枚举：`CITY_ADMIN`(地市管理员)、`COUNTY_ADMIN`(区县管理员) |
| `OrderStatus` | 工单状态枚举：待接单→方案制定→工单挂起→实施中→审核中→已归档 |
| `STATUS_MAP` | 状态标签映射（label + color） |
| `HiddenOrder` | 隐患工单接口（核心数据模型，含50+字段） |
| `ComplaintDetail` | 投诉工单明细接口（18个字段） |
| `HistoryRecord` | 历史进展记录接口 |
| `User` | 用户接口 |

### 4.2 工单状态流转

```
待接单(PENDING)
  ├── [接单] → 方案制定(PLANNING)
  └── [转派] → 仍为待接单，但处理人变更为区县管理员
        └── [接单] → 方案制定(PLANNING)

方案制定(PLANNING)
  ├── [提交方案] → 实施中(IMPLEMENTING)
  └── [挂起] → 工单挂起(HANGING)

工单挂起(HANGING)
  └── [解挂] → 方案制定(PLANNING)

实施中(IMPLEMENTING)
  └── [提交实施结果] → 审核中(AUDITING)

审核中(AUDITING)
  ├── [通过] → 已归档(ARCHIVED)
  └── [退回] → 实施中(IMPLEMENTING)

已归档(ARCHIVED)
  └── 终态，无操作
```

### 4.3 认证模块 (`src/stores/authStore.ts`)

- 使用 Zustand 管理登录状态
- 支持两种角色：地市管理员 / 区县管理员
- 登录时记录角色、地市、区县信息
- 提供 `login()` / `logout()` 方法

### 4.4 工单状态管理 (`src/stores/orderStore.ts`)

| 方法 | 说明 |
|------|------|
| `setFilters` | 设置筛选条件 |
| `resetFilters` | 重置筛选条件 |
| `setCurrentOrder` | 设置当前查看的工单 |
| `updateOrderStatus` | 更新工单状态 |
| `updateOrder` | 通用更新工单字段 |
| `submitPlan` | 提交方案 |
| `submitResult` | 提交实施结果 |

### 4.5 页面模块

#### Login.tsx - 登录页
- 角色切换（地市管理员 / 区县管理员）
- 地市/区县级联选择
- 密码输入（任意密码即可登录，无后端校验）

#### OrderList.tsx - 隐患工单列表页
- **统计卡片**：6种状态工单计数
- **筛选区**：地市、区县、状态、工单号、小区名称、日期范围、入库场景、是否超期
- **表格**：展示工单关键字段，支持分页（每页20条）
- **操作弹窗**：
  - 待接单：`接单`（所有角色）、`转派`（仅地市管理员）
  - 方案制定：`提交方案`、`挂起`
  - 工单挂起：`解挂`
  - 实施中：`提交实施结果`
  - 审核中：`通过`、`退回`

#### OrderDetail.tsx - 隐患工单详情页
- **工单信息**：关键摘要 + 更多信息 + 附件列表
- **处理进展**：操作按钮区 + 历史进展时间线
- **投诉关联**：展示关联的投诉工单明细
- 操作逻辑与列表页一致

#### ComplaintList.tsx - 投诉工单明细页
- 筛选：地市、区县、投诉时间、聚类工单号、家宽209号码、小区名称
- 超宽表格（最小1600px），支持横向滚动
- 展示18个投诉字段

### 4.6 转派功能

- **触发条件**：工单状态为"待接单" + 当前用户为地市管理员
- **行为**：将工单处理人变更为对应区县管理员，记录转派历史
- **权限**：仅地市管理员可见转派按钮

## 五、数据模型

### 5.1 隐患工单 (HiddenOrder) 关键字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 工单唯一标识 |
| `clusterOrderNo` | string | 聚类工单编号 |
| `city` / `county` | string | 地市 / 区县 |
| `cellName` | string | 小区名称 |
| `complaintCount` | number | 投诉数量 |
| `status` | OrderStatus | 工单状态 |
| `currentHandler` | string | 当前处理人 |
| `handlerRole` | UserRole | 处理人角色 |
| `dispatchTarget` | string | 派发目标 |
| `receiveTime` | string | 首次接单时间 |
| `faultOrderCount` | number | 故障工单数 |
| `complaintTop1/2` | string | 投诉TOP1/TOP2原因 |
| `broadbandCount` | number | 家宽数量 |
| `poorHomeNetCount` | number | 户内网质差客户数 |
| `poorAccessNetCount` | number | 接入网质差投诉客户数 |
| `planContent` | string | 方案内容 |
| `implementContent` | string | 实施内容 |
| `auditContent` | string | 审核内容 |
| `hangReason` | string | 挂起原因 |
| `historyProgress` | HistoryRecord[] | 历史进展记录 |
| `warehouseScene/Category/Form/Object` | string | 入库信息 |

### 5.2 Mock 数据规模

- 隐患工单：300条（每个状态50条）
- 投诉明细：500条
- 覆盖5个地市，每个地市多个区县

## 七、关键配置说明

### 7.1 Vite 配置 (`vite.config.ts`)

- 路径别名：`@` → `./src`
- 构建产物不生成 sourcemap（`hidden`）
- 优化依赖预构建

### 7.2 TypeScript 配置 (`tsconfig.json`)

- 目标：ES2020
- 模块：ESNext（bundler 模式）
- 路径别名：`@/*` → `./src/*`
- `strict: false`（宽松模式）

### 7.3 Tailwind 配置 (`tailwind.config.js`)

- 暗色模式：class 策略
- 扫描范围：`index.html` + `src/**/*.{js,ts,jsx,tsx}`

## 九、注意事项

1. **无后端**：当前项目为纯前端，所有数据来自 Mock，刷新页面后数据重置
2. **登录无校验**：任意密码即可登录，仅做角色和地市/区县选择
3. **废弃文件**：`Dashboard.作废.tsx`、`Home.tsx`、`index.html.作废` 为历史遗留，未使用
4. **主题切换**：`useTheme.ts` 已实现但未在项目中集成
5. **路径别名**：项目使用 `@/` 指向 `src/`，已在 Vite 和 TypeScript 中配置
